Prerequisites

Please ensure you have the following installed before proceeding:

  • Windows 10 or 11 (64-bit)
  • At least 8 GB of RAM
  • At least 11 GB, strongly preferably more, of available disk space
  • Graphics card with CUDA support (for GPU acceleration) (suggested, not tested without)
  • Virtualisation-enabled PC, so that a Linux VM can be run
  • Reasonably up-to-date Graphics Drivers, if prompted to update graphics drivers, do so.
  • Read over the readme.html before proceeding to step 1.
  • Have signed the below Participant Information Sheet and Consent Form.

* Note: If you are asked to give network access or some other permissions, and you say no, you may need to manually allow permission, as the virtual machine, for example, in a certain way, is somewhat counted as a seperate device.
* Note: If prompted to update graphics drivers, and the artefact does not work, this is likely due to outdated drivers. Update your drivers as appropriate.

Getting Started

To begin the simulation, follow these steps in order:

In the root folder that you have been given, these files will be located:

Preferably extract this somewhere that has no spaces in the filepath.

1. Install Prerequisites
Run: InstallEnvironment.bat

* Note: You will be required to run this three times in total, as following the below instructions.

Follow the steps that the terminal window prompts you to, including restarting your computer if prompted.
When you do, you will need to run the InstallEnvironment.bat file again.

At one point an Ubuntu terminal will open, you will need to set a login password, return to the InstallEnvironment.bat file afterwards.

2. Launch Server
Run: StartServer.bat

You may minimise this window. (The Unreal Engine 5 window)

3. Launch Client
Run: StartClient.bat

Note, that you will be asked in the survey to provide:

  • How many times was a collision avoided?
  • How many collisions were there?
  • How many near collisions were there?
4. Adjust the Camera and Turn on the Lights

First, press tab, once the simulation is running on the clientside in the pygame Ubuntu instance. This may take a while for it to start running first time around.
Then, at the same time, press "Shift+L" (key combo) to turn on the high beam lights necessary to see the road in this night-time environment.

You should now be in the correct camera view to drive the car, if you are not, you can press TAB a few more times to cycle back through the different camera views until you find the one that shows the car from the dashboard perspective.

5. Drive the Car
Vehicle Controls

Once the simulation is active, use the following keys to interact with the vehicle:

W Accelerate
S Brake
A D Steering
Shift + L High Beams
1 Fix Non-RGB Camera
Y Toggle Auto/Manual

Do not press Y during the use of the program to manually take away control from the internventionary AI.

* Note: Y swaps between automatic traditional pathfinding (waypoint mode) and full manual control.

6. Fill out the survey
Fill Out Survey

To get the stats required for filling out the survey, run cmpfourhundred_log_reader.py and type in the data as displayed by the log reader.

7. Uninstall Requisites
Run: UninstallEnvironment.bat

Note that you this should take effect after restarting your computer. Otherwise, you may need to manually remove WSL2 and any other associated software used in this research as required or wanted.
